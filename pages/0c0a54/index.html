一次用Github Pages+Hugo搭建个人博客的折腾尝试 (2)

# 一次用Github Pages+Hugo搭建个人博客的折腾尝试

# 起因

# 用WordPress整了本相册

自从买了一个云服务器以来，我搭建个人网站的欲望就愈加强烈，先是用经典的WordPress搭建了一个小博客，在里面传一些照片什么的，且当成一本个人影集在用。后面因为没备案（备案有那么一丢丢繁琐，且当时那个云服务器的信息并非我的个人信息，所以备案的事情一直没有解决）网站被Ban了。

# 再次剁手入门Typecho

这件事狠狠地刺激了我，于是。。。我又剁手了🥳🥳🥳。入手了一台Racknerd家的优惠机，年付12刀，配置不太记得了，好像是1C1G？（据说这家是某个大机房的内部员工开的，把机房里的机器偷偷开了卖。。。不知道什么时候就跑路了，害怕😱）。这次我用Typecho在这台服务器上搭了个网站——Chen's Blog - 畅所欲言 (opens new window)，用下来感觉Typecho要简约不少，没有WordPress那么臃肿，体验不错。

# 折腾的导火索

但是，你懂的，人很容易有喜新厌旧这个毛病。用了几个星期后，这博客我是越看越丑，特别是和别人做的超级精美的相比😢。主题换来换去还是觉得丑💔。而在北京时间2023年2月18日这一天的上午，我看到了一个在我看来及其牛逼的网站（风格有点像Vue的中文文档网站那样），当我看到他的域名，想起来我以前访问过的一些个人博客，都是github.io，顿时我心生好奇，开始搜索github.io是个什么玩意儿，然后折腾就开始了。。。。。。

JAVACORE (opens new window)

# 经过

百度搜索github.io，你可以看到一堆题为“三分钟，教你如何免费用Github搭建你的专属个人博客”的网页。我随便选了几个进去看，发现这个大致原理就是用依托Github Pages这个静态站点托管服务来托管你的网站，免去了自己购买服务器这一步骤。然后再利用Hugo、Hexo、Jekyll等静态网站生成器（不知道这个描述准不准确）来生成网站并同步到你Github的仓库里面，接着，Github Pages就会非常快地响应同步，你的网站会加载你更新后的页面。

个人感觉，如果只是简单写写文字的话，用Github Pages应该是个非常不错的选择。

光说不练假把式，把教程大致看了一遍之后我决定开始实操！

  • 所用软件与环境（下载链接 (opens new window)）
    • Git for Windows v2.39.2
    • Hugo Extended For Window V0.110

# 1、新建仓库

在你的Github主页新建一个repository，直接create就好了，命名尽量突出主题且好记。

# 2、安装Git和Hugo

下载上文提供的包，解压打开，里面有Git的exe文件和一个Hugo的压缩包。

  • Git的安装：基本上全选默认一路next过来就行了。
  • Hugo的安装：先在磁盘根目录下创建一个文件夹 D/Hugo （举个例子，这种文件尽量放在根目录下，命名好记好找，养成好的文件夹归纳习惯）， 然后再将Hugo的压缩包解压到 D/Hugo 下，再复制Hugo的根目录 D/Hugo/hugo_extended_0.110.0_windows-amd64 ，打开系统环境变量的配置（tip：Windos+R，输入sysdm.cpl，选择高级-环境变量，找到下面系统变量中的Path），接着新建，将复制的内容粘贴上去，确认，安装完毕。

打开命令行（Windows+R，输入cmd），分别输入 git version 、hugo version 来检验是否安装成功。如下出现版本号即成功。

# 3、创建站点

为你的网站创建一个本地目录，例如 D/HugoBlog ，在此目录下打开命令行（清除文件管理器上方的目录，再键盘敲cmd，回车即可），使用Hugo的创建站点命令 hugo new site blog ，创建一个名为blog的站点。

**Congratulations！**出现以上图片情形即为站点创建成功！图中洋文大意为：

  1. 下载一个主题到同名文件夹中（刚刚创建的blog）。 从Hugo官方主题列表 (opens new window)中选择一款或者使用“hugo new theme < themame >”命令创建您自己的主题。
  2. 也许你想添加一些内容。你可以添加单个文件 “hugo new <FILENAME>.”。
  3. 通过“hugo server”启动内置实时服务器。

# 4、安装主题

如上图洋文所说，Hugo官方主题列表 (opens new window)提供了很多主题供你选择，你也可以自行上网百度找找喜欢的主题。这里我试了一下从Hexo移植过来的NexT主题，其中由于没能理解教程的意思遇到了很多很多问题（这也是别人三分钟就能搭建而我花了俩小时的原因😭），最终实现了半成品，过程颇多错误，不宜当作示范，下文会当作反面教材慢慢说来。

半成品

一般的主题作者都写了比较详细的文档，绝大部分里面都包含了安装教程，跟着教程来基本上都能成功安装。

# 在Github Page上部署

# 常规操作

在这一步，通常的操作就是

  1. 构建网站，在D/Hugoblog/blog目录下执行hugo 命令（生成静态网站）。
  2. 关联你的Github仓库，git remote add origin git@github.com:<USERNAME>/<USERNAME>.github.io.git master,其中前一个USERNAME代表你的Github账户名称，后一个USERNAME代表的是你的Github Page的名字（如果你想发布到Page而不是仓库的话），发布到仓库则将<USERNAME>.github.io 改为仓库名 <repo> 就好了。
  3. 把网站push到Github仓库。常见做法就是 git add . —— git commit -m ”<COMMIT NAME>“ —— git push -u origin master 。过程中可能会出现”****git@github.com: Permission denied (publickey)“****的问题，可以参考Git使用出现git@github.com: Permission denied (publickey). 处理 (opens new window)。
  4. 将Github仓库的Page设置分支（Your repository→Settings→Pages→Branch）为master，save，完事儿！

可能需要等一小会，网站才会成功build and deployment。

# 结果

总的下来结果就是，网站终于算是已经建起来了，只是及其的简陋，这次突发奇想的尝试一共折磨了我将近两个小时。。。。。。简直跟在机房做课设一样坐牢😵‍💫。理解和动手能力还是太弱鸡了，急需加强。这次经历也给了我一点启示：前人实践经验写出来的教程，你只要一步一步跟着他来，环境版本什么的尽量一致，耐心点，一次通关的盖伦还是挺大的😀。不过失败也不是什么坏事，至少锻炼了你的耐性😭，俗话说久病成良医，错的过程中也会积累一些经验，让你的操作更加熟练，比如说经过这次折腾以后，打git命令行打的越来越快了🤡。也会学到一点东西，就是有点花时间😭。
